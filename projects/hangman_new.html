Hangman

In this project, students will learn how to build a basic hangman game that can be played in the terminal.

The game starts by asking a player 1 to type a word to guess. A second player plays the game by typing a letter. The computer tells player 2 whether the letter is in the word or not, and displays 1) the right letters guessed so far and 2) the wrong letters guessed so far. The game ends after player 2 runs out of tries or guesses the complete word.

Learning Goals:

  • Practice the use of conditionals
  • Practice the use of arrays
  • Practice the use of iteration
  • Review a couple string methods
  • A small dip into defining methods

Next, work together with the group to first write pseudocode for the game. See here for key concepts you want to touch on.

Add-on's

Here are some extensions the students can work on if they finish early or want to spend more time on their project:

  1. Not allowing the player to play a letter already guessed
  2. Not allowing the player to type something invalid (for example a number or a full word)
  3. Allowing the player to set the difficulty by choosing how many tries they get.
  4. Making the difficulty automatic based on the number of letters in the word. (let the students think it through: what should be the proportion between number of tries, and length of the word?)
  5. Instead of asking the user for a word, load it randomly from an array of predefined words.
  6. Adding another loop so the game can be played multiple times.
  7. Go back to their hi-lo and rock, paper, scissors games and add loops.
