Concept Review

Here is a list of all the programming concepts CodeNow students get introduced to during our beginning workshop. Each section includes a link to mini exercises students can complete in IRB to help them further understand the concepts. The list is in alphabetical order.

Arrays

An array holds a collection of items. You can think of it as a list. In Ruby, the types of items in the array is not important. For example, a 3 item array can hold an integer, a string, and another array. However, the order of items is important.

Arrays are zero-indexed. Its elements can be accessed by using the bracket operators [n], where n is the index or position of the element we are trying to access. Since this is an introductory workshop, do not worry about this too much.

However, often convenience methods, such as “first”, “last”, “length”, and “sample” are used to access the elements of an array

Boolean Logic

Conditionals and looping routines both contain expressions that can evaluate to “false” or “true”. Also, both can include one expression to be evaluated or a combination of expressions. When using a combination of expressions, we use boolean operators.

The main boolean operations are:

  • AND: denoted by &&
  • OR: denoted by ||
  • NOT: denoted by !

The rules for combining expressions using each boolean operator are as follows:

Conditionals

Conditionals are a control structure. Using them allow us to execute code we want to execute only when certain conditions are met.

Conditionals are important to program control flow.

Remember that code is executed from top to bottom. In cases when we want for some code to execute only under certain conditions, we use conditionals. If the conditions are not met, the code following those conditions will be skipped.

There are four components to a conditional block in Ruby:

  1. if (required)
  2. elsif (optional)
  3. else (optional, does not require an expression to be evaluated)
  4. end (required)

It is important to understand that the if/elsif/else statements will operate sequentially and will exit the if/elsif/else as soon as one condition is satisfied.

Often conditionals are based on expressions that compare values. There are six operators used when doing value comparisons:

  • == equals
  • != is not equal to
  • < is less than
  • > is greater than
  • <= is less than or equal to
  • >= is greater than or equal to

Data Types

The data that we store and use in our programs can be of different types. Two main types are string and integer.

Numeric data that are whole numbers are of type integer, hence the variables we use to store it are also of type integer.

Among the operations we can use with integers are the following: addition, subtraction, multiplication, and division.

To perform these operations we use the following symbols: + for addition, - for subtraction, * for multiplication, and / for division.

Textual data is of type string, hence the variables we use to store it are also of type string. Strings are written inside double quotes.

We can also perform “operations” on strings. In order to perform “operations” on strings we use methods. Some of the methods we can use are:

  • puts = when we want to print something to the screen.
  • gets = when we want to capture user input.
  • .chomp = when we need to strip the new line character.
  • .downcase = when we want to convert all characters to lowercase.
  • .upcase = when we want to convert all characters to uppercase.
  • .capitalize = when we want to convert the first character to uppercase and the remainder to lowercase
