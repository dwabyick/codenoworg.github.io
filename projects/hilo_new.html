Hi-Lo Guessing Game

In this project, students will build a procedural version of the Hi-Lo Guessing Game.

Learning Goals:

  • Understand program control flow and the process of writing pseudocode.
  • Practice fundamentals of Ruby including capturing and using user input (puts/gets) and using variables (strings, integers) and control structures (conditionals: if/end)

Step 1 – Writing pseudocode

Work together with the group to first write pseudocode for the Hi-Lo game. See here for key concepts you want to touch on.

  1. As a group discuss what actions need to happen during the game and when should they take place.
  2. Explain that the computer runs code sequentially, starting from the first line of the program and ending at the last line.
  3. As a group, come up with the first two to three lines of pseudocode.
  4. Ask the students to continue the discussion with their partners and complete their pseudocode

Have the students run the program to prove that the computer does not execute comments.

Step 2 – Setting variables

Introduce students to variables (strings and integers). See here for key concepts you want to touch on.

  1. Go over the purpose for which we would want to store information in our game, the scenarios during which we would want to manipulate the information (if any), the place in the program where we would want to use the information stored, etc.
  2. Ask student to add variables to their game using their pseudocode as a guide.
    • Tell students to start off by setting the variables to a fixed value.

Have students run their programs to ensure that their code is in a working state.

Step 3 – Getting user input

Give students a brief introduction to the data types integer and string. See here for key concepts you want to touch on.

  1. Ask students to add player input to their game using their pseudocode as a guide
    • Remind students that all user input is always captured as a string, and we can use “.to_i” if we need the numerical value.

Have students run their programs to ensure that their code is in a working state.

Step 4 – Conditional statements

Introduce students to conditionals (“if/end” only). See here for key concepts you want to touch on.

  1. Ask student to add conditionals to their game using their pseudocode as a guide
    • Remind students of the difference between using = and using ==

Have the students run their program to confirm it works and test all three cases by trying different combinations of numbers

Step 5 – Adding randomness

  1. Explain to students that we can use the rand() method to generate a random number.
  2. Point out the need to use +1 (e.g., rand(10) + 1), and explain why.
  3. Ask students to use rand() in their game

Have students play their game and prove that the secret number is no longer fixed. Since the secret number is random, students should have difficulty proving all three cases. Show them how they can debug their programs by printing out the secret number using puts.

Step 6 – Let's review

Now that you have completed the base project, do a quick review of the concepts with your students.

  • Ask students to tell you, without looking at their notes, what are the concepts they learned about in this project. Students should mention the building blocks introduced and/or reviewed in the project:
    • pseudocode, program control flow, capturing and using user input (puts/gets), using variables (strings, integers), control structures (conditionals: if/end)
  • If students do not mention some of the building blocks, ask them about them specifically. See here for sample questions.
  • Finally, ask students if they would like you to review any of the concepts or if they have any additional questions.

Step 7 – Add-on's

After completing the base project and doing a quick review of the building blocks introduced and/or reviewed, work with your students on adding additional features to the project. Work on add-on’s only if the time allocated to this project has not ran out.

Suggestions for add-on’s

  1. Add difficulty levels. For example, in the easy level the player has to guess a number between 1 and 10; in the medium level, between 1 and 50; in the hard level, between 1 and 100; etc.
  2. Add version options. For example, the player can choose to play the version where the number guessed is a multiple of 5.
  3. Add a “quit” option, where the player is asked if he/she wants to play or quit.
  4. Let students come up with their own ideas on how to extend their programs - see what they come up with!
