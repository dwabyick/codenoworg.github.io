Hi-Lo Guessing Game

In this project, students will build a procedural version of the Hi-Lo Guessing Game.

Learning Goals:

  • Practice the fundamentals of Ruby including setting variables, getting user input, printing messages
  • Understand control flow and conditional branching

The design decisions in the following code are intentional to help absolute beginners quickly learn the basics without overwhelming them with intermediate concepts.

This tutorial is open source. If you notice errors, typos, or have questions/suggestions, please submit them to the project on Github.

Step 1 – Writing pseudocode

Work together with the group to first write pseudocode for the Hi-Lo game. See here for key concepts you want to touch on.

  1. As a group discuss what actions need to happen during the game and when should they take place.
  2. Explain that the computer runs code sequentially, starting from the first line of the program and ending at the last line.
  3. As a group, come up with the first two to three lines of pseudocode.
  4. Ask the students to continue the discussion with their partners and complete their pseudocode
  5. Have the students run the program to prove that the computer does not execute comments.

Step 2 – Setting variables

Introduce students to variables (strings and integers). See here for key concepts you want to touch on.

  1. Go over the purpose for which we would want to store information in our game, the scenarios during which we would want to manipulate the information (if any), the place in the program where we would want to use the information stored, etc.
  2. Ask student to add variables to their game using their pseudocode as a guide.
  3. Tell students to start off by setting the variables to a fixed value.
  4. Have students run their programs to ensure that their code is in a working state.

Step 3 – Getting user input

Give students a brief introduction to the data types integer and string. See here for key concepts you want to touch on.

  1. Ask students to add player input to their game using their pseudocode as a guide
  2. Remind students that all user input is always captured as a string, and we can use “.to_i” if we need the numerical value.
  3. Have students run their programs to ensure that their code is in a working state.

Step 4 – Conditional statement

Introduce students to conditionals (“if/end” only). See here for key concepts you want to touch on.

  1. Ask student to add conditionals to their game using their pseudocode as a guide
  2. Remind students of the difference between using == and using ==
  3. Have the students run their program to confirm it works and test all three cases by trying different combinations of numbers

Step 5 – Adding randomness

Have students play their game and prove that the secret number is no longer fixed. Since the secret number is random, students should have difficulty proving all three cases. Show them how they can debug their programs by printing out the secret number using puts.
